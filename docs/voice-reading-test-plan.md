# 语音朗读测试方案

| 项目 | 内容 |
|---|---|
| 版本 | `0.2 Draft` |
| 被测范围 | Markdown 本地语音朗读、后台播放、缓存与进度恢复 |
| 核心门禁 | 离线、隐私、中文可懂度、60 分钟后台稳定性 |

## 1. 测试目标

验证语音朗读在不同 Android 版本、厂商系统和本地 TTS 引擎下能够：

- 正确提取并自然朗读 Markdown；
- 在不上传正文的前提下离线工作；
- 快速开始、连续播放并正确恢复进度；
- 在后台、锁屏及外部媒体控制下稳定运行；
- 面对缺少音色、合成失败、空间不足和进程回收时可理解、可恢复。

## 2. 主要风险

| 风险 | 影响 | 测试重点 |
|---|---|---|
| 厂商 TTS 输出格式或回调行为差异 | 无法播放、卡在准备中 | 多引擎真机合成、文件校验、超时与重试 |
| 中文数字和中英混排误读 | 内容难以理解 | 专项语料、发音词典和主观听测 |
| 后台限制或音频焦点处理错误 | 锁屏中断、与其他音频冲突 | API 版本和国产 ROM 长时测试 |
| 合成速度跟不上播放 | 段间卡顿 | 慢速设备、长段、2× 倍速和预取测试 |
| 私有正文进入日志/公共目录 | 隐私泄露 | 网络、日志、备份和文件系统审计 |
| 文档更新后缓存错配 | 播放旧内容或恢复错位 | 内容哈希与缓存失效测试 |

## 3. 测试层次

### 3.1 单元测试

- Markdown AST 到 `SpeechDocument` 的映射；
- 标题、段落、列表、引用、链接、图片、代码、表格和 Front matter 规则；
- 标点分段、超长段落和 TTS 最大输入长度边界；
- 百分比、金额、日期、时间、单位、序号及中英混排规范化；
- `AIDC`、`EBITDA` 等词典优先级和词边界；
- `documentId`、`contentHash`、段落 ID 和缓存键稳定性；
- 播放状态机、重试、取消、迟到回调及进度节流保存；
- LRU 清理不会删除当前播放或正在生成的文件。

所有文本规则采用表驱动测试和固定快照。不得只断言“非空”，应断言最终交给 TTS 的完整文本。

### 3.2 组件与集成测试

- 使用 Fake `SpeechProvider` 生成确定性短音频，验证合成队列和 Media3 播放队列；
- 前两段优先、后续预取、失败跳过、切换文章取消任务；
- 临时文件原子转正，损坏或零字节文件不得命中缓存；
- 内容变化、音色变化和规则版本变化分别触发正确失效；
- DataStore 持久化及进程重建后的状态恢复；
- MediaController 断开和重连时 UI 状态一致。

### 3.3 真机端到端测试

真实系统 TTS 只在设备测试中使用，验证：

- 枚举并筛选离线中文音色；
- 合成文件非空且 Media3 可播放；
- 通知栏、锁屏和耳机控制；
- 屏幕关闭、切换应用、划掉最近任务和系统回收；
- 飞行模式和无网络环境；
- 电话、导航、音乐、闹钟以及耳机断开时的音频焦点行为。

## 4. 环境矩阵

最低覆盖：

| 维度 | 配置 |
|---|---|
| Android | API 28、31、33、35/36 |
| 设备 | AOSP/Pixel 类设备；至少一台当前项目使用的小米或其他国产 ROM 真机 |
| TTS | Google/系统默认引擎；可获得时增加一个厂商或第三方离线引擎 |
| 网络 | Wi-Fi、移动网络、断网、飞行模式 |
| 音频输出 | 扬声器、有线耳机、蓝牙耳机 |
| 电源状态 | 充电、低电量、省电模式、Doze |
| 文档规模 | 1 分钟短文、30 分钟常规文章、2 小时超长文章 |

API 35/36 是发布门禁环境，因为音频焦点和前台服务限制更严格。国产 ROM 必须验证锁屏后 60 分钟连续播放。

## 5. 测试语料

建立不含真实私有数据的固定语料库：

1. Markdown 全元素文档：六级标题、列表、引用、链接、图片、代码和表格。
2. 中文数字文档：`11.1%`、`908亿元`、`2026-07-22`、`09:30`、负数和范围。
3. 金融科技术语文档：AIDC、EBITDA、AI、5G、GitHub、API 及股票代码。
4. 中英混排文档：中文句子中的英文公司名、产品名和缩写。
5. 异常文档：超长段落、连续符号、emoji、空表格、超长 URL 和非法 Unicode。
6. 性能长文：至少 30 分钟和 2 小时的可朗读内容。

主观听测使用同一份版本化语料和同一基准音色，避免引擎更新导致结果不可比较。

## 6. 功能用例

### 6.1 首次播放与控制

| 编号 | 场景 | 预期结果 |
|---|---|---|
| VR-001 | 首次点击“听全文” | 显示准备状态，前两段生成后开播，不等待全文 |
| VR-002 | 暂停后继续 | 从原段内位置继续，误差不超过 1 秒 |
| VR-003 | 前后 15 秒 | 跨段时位置正确，无重复或漏读 |
| VR-004 | 上一段/下一段 | 跳到目标段开头，播放器状态正确 |
| VR-005 | 切换倍速 | 立即生效，不触发重新合成，音调无异常 |
| VR-006 | 拖动全文进度 | 落到正确段及段内时间；未生成段优先合成后播放 |
| VR-007 | 点击正文段落 | 从该段开始朗读 |
| VR-008 | 播放完成 | 状态变为完成，再播放从开头开始 |

### 6.2 音色与离线

| 编号 | 场景 | 预期结果 |
|---|---|---|
| VR-020 | 音色列表 | 只把无需网络的中文音色作为默认可选项 |
| VR-021 | 切换音色并试听 | 试听正确，新音色使用独立缓存 |
| VR-022 | 无中文离线音色 | 不崩溃、不偷偷联网，展示安装指引 |
| VR-023 | 飞行模式首次合成 | 已安装离线音色可正常生成和播放 |
| VR-024 | 系统切换 TTS 引擎 | 重新检测能力，失效不可兼容任务并给出状态 |

### 6.3 Markdown 与正文联动

| 编号 | 场景 | 预期结果 |
|---|---|---|
| VR-030 | 链接和图片 | 读链接标签、不读 URL；图片按设置跳过或读替代文字 |
| VR-031 | 代码块 | 按设置跳过，Markdown 符号不被朗读 |
| VR-032 | 表格摘要 | 正确读行列数，不读竖线分隔符 |
| VR-033 | 表格逐行 | 列名与单元格对应，无空白噪声 |
| VR-034 | 阅读与播放解耦 | 朗读中手动滚动、搜索或翻页，正文不被自动定位或滚动 |
| VR-035 | 重开恢复 | 重新打开文章后，恢复最近播放进度但不改变当前阅读位置 |

### 6.4 缓存与恢复

| 编号 | 场景 | 预期结果 |
|---|---|---|
| VR-040 | 再次播放同一文章 | 命中缓存，快速开播 |
| VR-041 | 修改正文后同步 | 不播放旧缓存，按新内容重建 |
| VR-042 | 强制停止后重开 | 提供继续朗读，段落正确、误差不超过 3 秒 |
| VR-043 | 缓存达到上限 | 回收最旧内容，不删除活动文件，总量回落到阈值内 |
| VR-044 | 合成中存储不足 | 清理后重试；仍失败时明确提示且应用可继续阅读 |
| VR-045 | 缓存文件损坏 | 判为未命中并重新合成，不把异常传递为崩溃 |

## 7. 后台与媒体测试

每个场景至少在 API 28、33、35/36 各执行一次：

- 锁屏后连续播放 60 分钟；
- 回桌面、打开其他应用、返回 RepoLeaf；
- 从最近任务划掉 RepoLeaf；
- 通知栏和锁屏播放、暂停、上一段、下一段；
- 蓝牙耳机单击播放/暂停及切换；
- 断开耳机，必须暂停且不得转为扬声器外放；
- 播放音乐后启动朗读，音乐按系统策略暂停；
- 朗读时启动导航语音、电话和闹钟，焦点行为符合语音内容策略；
- 进入省电模式和 Doze；
- 暂停超过 10 分钟后服务退出，进度仍可恢复；
- 系统杀死服务后不自动外放，重新打开应用可继续。

测试记录必须包含设备、系统版本、TTS 引擎及版本、音色、是否离线和电池优化状态。

## 8. 性能与稳定性

基准中端真机上的目标：

| 指标 | 门槛 |
|---|---|
| 缓存命中首次出声 | P95 ≤ 2 秒 |
| 冷启动且需合成首次出声 | P95 ≤ 5 秒 |
| 播放控制响应 | P95 ≤ 150 ms |
| 常规文章连续播放 | 30 分钟无可感知段间卡顿 |
| 后台稳定性 | 60 分钟无非预期停止 |
| 进度恢复误差 | ≤ 3 秒 |
| 缓存上限 | 清理完成后不超过配置值的 105% |
| 内存 | 2 小时播放无持续线性增长，无 OOM |

额外执行 2× 倍速加慢速 TTS 合成的压力测试。若预取跟不上，应显示短暂缓冲状态，不得重复、乱序或跳过正文。

## 9. 中文朗读质量

由至少 3 名中文母语测试者在基准离线音色上盲听固定语料，按 1–5 分评价：

- 可懂度；
- 停顿自然度；
- 数字、金额和日期正确性；
- 中英混排流畅度；
- 长时间收听舒适度。

发布要求：平均可懂度不低于 4 分；数字、金额和日期不得出现改变事实含义的 P0/P1 错读；高频术语问题必须通过规范化或词典修复。音色本身的音质差异单独记录，不将第三方引擎固有限制误判为应用缺陷。

## 10. 隐私与安全测试

- 使用代理和系统网络统计确认朗读过程中无应用新增 TTS 网络请求；
- 在飞行模式执行完整核心流程；
- 检查 Logcat、崩溃文件和诊断导出，不得包含正文、Token、私有仓库路径或音频；
- 检查音频只存在于应用私有缓存，不出现在系统音乐库、相册或下载目录；
- 验证 Android 备份规则排除语音缓存和敏感播放上下文；
- 验证选择声明需要网络的音色时不会被默认使用或静默回退；
- 清除朗读缓存后，音频文件实际删除且数据库引用同步清理。

## 11. 自动化与缺陷分级

建议 CI 门禁：

1. JVM 单元测试和规范化快照；
2. Android 仪器测试：状态存储、服务连接和缓存；
3. Fake TTS 端到端测试：从点击入口到 Media3 播放；
4. 静态检查、资源和 Manifest 权限检查；
5. 每晚或发布前运行真机设备池测试。

缺陷优先级：

- `P0`：私有内容外泄、持续崩溃、无法停止外放；
- `P1`：后台播放不可用、内容乱序/漏读、事实含义被错误朗读、进度丢失；
- `P2`：特定格式或设备体验明显下降但可绕过；
- `P3`：视觉、提示文案和低频兼容问题。

## 12. 发布门禁

全部满足后才可发布：

- 自动化测试全部通过，无 P0/P1 未解决缺陷；
- API 28、33、35/36 核心用例通过；
- 至少一台国产 ROM 真机锁屏后台播放 60 分钟通过；
- 基准 TTS 引擎中文质量达到第 9 节要求；
- 飞行模式下首次合成和缓存播放均通过；
- 通知栏、锁屏、蓝牙和音频焦点用例通过；
- 文档更新、进程重建、缓存损坏和空间不足恢复用例通过；
- 隐私审计确认正文和音频未离开本机私有空间；
- 首次出声、控制延迟、内存和缓存指标达到门槛。
