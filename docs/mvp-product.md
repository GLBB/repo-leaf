# RepoLeaf MVP 产品方案

版本：`0.1`  
状态：Draft  
目标平台：Android 9（API 28）及以上

## 1. 产品定义

RepoLeaf 是一个面向 Android 的多仓库知识阅读器。用户可以配置多个 GitHub 仓库，将仓库资料同步到手机，并在离线状态下继续阅读。首个 MVP 聚焦 Markdown：先把目录、链接、图片、代码、主题和阅读进度做好，再扩展 HTML、PDF 和 XLSX 内置阅读。

MVP 的核心价值不是在手机上编辑 Git，而是让“存放在 Git 仓库中的资料”具备接近原生阅读应用的浏览、同步和阅读体验。

## 2. 目标用户

- 使用 GitHub 保存团队文档、产品资料或技术知识库的研发团队。
- 使用 Markdown 和 Git 管理个人知识库的开发者。
- 需要在手机上离线查看项目手册、Wiki 和技术资料的团队成员。

## 3. MVP 目标

用户应能在 5 分钟内完成以下闭环：

1. 添加一个或多个 GitHub 仓库。
2. 将指定分支或子目录同步到本地。
3. 按仓库或“全部资料”浏览文档。
4. 获得完整、流畅的 Markdown 阅读体验。
5. 收藏资料并在下次打开时恢复阅读位置。
6. 断网后继续阅读已同步内容。

## 4. MVP 范围

### 4.1 必须包含

| 能力 | MVP 定义 |
|---|---|
| 多仓库 | 最多配置 10 个仓库；仓库名称、地址、分支、资料子目录相互独立 |
| Git 平台 | GitHub 公共和私有仓库；通过 RepoLeaf GitHub App 以只读权限访问用户明确选择的仓库 |
| 同步 | 手动全量快照同步；基于远端 commit SHA 判断是否需要下载 |
| 离线 | 同步成功后，文件和索引保存在应用私有目录，无网络可正常阅读 |
| 浏览 | “全部资料”和单仓库两种入口；目录树、最近阅读、收藏 |
| 搜索 | 文件名、相对路径和文档标题的跨仓库搜索 |
| Markdown | 标题、目录、表格、任务列表、引用、代码块、图片、相对链接、锚点、深浅主题和字体缩放 |
| 其他文件 | HTML、PDF、XLSX 可在目录中识别并显示元数据；MVP 不承诺内置渲染，可通过系统应用打开 |
| 阅读状态 | 最近阅读、收藏、每个文件的页码或滚动位置 |
| 错误恢复 | 单个仓库失败不影响其他仓库；保留上一次可用本地副本 |

### 4.2 明确不包含

- 手机端修改、提交、合并或解决 Git 冲突。
- GitLab、Gitea 和任意 Git 服务的正式支持；MVP 只预留 Provider 接口。
- SSH Key、Git LFS 和 Submodule。
- HTML、PDF、XLSX 的内置阅读器，以及 Office 编辑、公式重算、宏和图表。
- Markdown 全文索引、OCR、AI 问答和多人协作。
- iOS、桌面端和平板专用双栏布局。

## 6. 核心用户流程

### 6.1 添加公共仓库

1. 点击“添加知识库”。
2. 输入 GitHub 仓库 URL，例如 `https://github.com/org/docs`。
3. 应用读取仓库信息并提供默认分支。
4. 用户可选资料子目录并设置显示名称。
5. 点击“保存并同步”。
6. 同步成功后进入仓库目录。

### 6.2 连接 GitHub 和添加私有仓库

1. 用户点击“连接 GitHub”，应用启动 GitHub App Device Flow。
2. 应用显示一次性验证码并打开 GitHub 授权页面。
3. 用户授权 RepoLeaf，并为 GitHub App 选择允许访问的个人或组织仓库。
4. 应用只展示用户有权限且已授权给 RepoLeaf 的仓库。
5. 用户选择一个私有仓库、分支和资料子目录，保存并同步。
6. 授权失败、过期、被撤销或组织 SSO 未激活时，应用保留已有离线资料并给出重新连接入口。

### 6.3 同步仓库

1. 用户下拉刷新单个仓库，或在首页触发“同步全部”。
2. 应用先读取远端默认分支最新 commit SHA。
3. SHA 未变化时直接报告“已是最新”。
4. SHA 变化时下载并校验仓库快照，在临时目录建立索引。
5. 成功后原子替换本地版本；失败时继续使用旧版本。

### 6.4 阅读和恢复

1. 用户从目录、搜索、最近阅读或收藏打开资料。
2. Markdown 使用内置渲染器；其他已识别文件可选择系统应用打开。
3. 退出阅读页时保存进度。
4. 再次打开时恢复到上一次位置。

## 7. 功能需求

### 7.1 仓库管理

- 仓库唯一标识使用本地 UUID，不以仓库名称作为主键。
- 支持修改显示名称、分支和资料子目录。
- 支持停用仓库；停用后不参与“同步全部”和聚合搜索。
- 删除仓库前显示其本地占用空间，并明确提示将删除离线文件和相关阅读记录。
- 相同远端仓库允许配置不同分支或不同子目录。

### 7.2 文档识别

- 支持扩展名：`.md`、`.markdown`、`.html`、`.htm`、`.pdf`、`.xlsx`。
- 忽略隐藏目录、`.git`、可配置的排除路径和不支持的二进制文件。
- 文档唯一标识为 `repositoryId + relativePath`，避免跨仓库同名冲突。
- Markdown 标题优先取 YAML front matter 的 `title`、第一个一级标题，最后回退到文件名。

### 7.3 状态反馈

- 每个仓库显示：从未同步、同步中、已同步、已过期、失败。
- 显示最后成功同步时间和当前本地 commit 短 SHA。
- 错误提示区分网络、鉴权、仓库不存在、空间不足、文件损坏和格式不支持。
- 长时间任务必须可取消，不得阻塞其他仓库的本地阅读。

### 7.4 GitHub 账号与私有仓库

- MVP 至少支持连接一个 GitHub 账号。
- Release 版本以 GitHub App Device Flow 为正式认证方式，不要求用户粘贴 classic PAT。
- GitHub App 仓库权限限定为 `Contents: read-only`；不申请提交、Issue、PR、Actions 或管理权限。
- 用户可以只授权选定仓库，不必开放全部私有仓库。
- 断开账号会删除本地凭证，但不会静默删除已同步资料；删除离线资料需要单独确认。
- 私有仓库名称、路径和内容不得进入遥测、崩溃报告或系统公共存储。

## 8. 体验与非功能指标

| 指标 | MVP 目标 |
|---|---|
| 冷启动 | 中端 Android 手机首次可交互时间小于 2.5 秒 |
| 列表响应 | 1 万条文档索引下，滚动无明显卡顿，搜索反馈小于 500 ms |
| 阅读打开 | 1 MB Markdown 小于 1 秒进入可阅读状态；5 MB Markdown 小于 2 秒 |
| 稳定性 | 阻断级崩溃为 0；同步失败不破坏已有离线副本 |
| 存储 | 在仓库详情中展示本地占用；空间不足时同步前中止 |
| 无障碍 | 主要按钮有语义标签；正文支持系统字体缩放和深色模式 |

## 9. MVP 验收标准

MVP 可发布需要同时满足：

- 可添加至少 3 个 GitHub 仓库，并独立切换、同步和删除。
- 公共和私有仓库均能完成同步；访问令牌和刷新令牌不出现在日志、明文数据库或系统备份中。
- GitHub App 仅有 Contents 只读权限，未获授权的私有仓库不可见、不可同步。
- Markdown 基线库中的语法、相对图片、跨文件链接和锚点跳转全部通过。
- HTML、PDF、XLSX 能被识别，不会误解析或导致崩溃，并可交给兼容的系统应用打开。
- 飞行模式下可阅读全部已同步资料，收藏和阅读进度正常保存。
- 某个仓库同步失败时，其他仓库和该仓库旧版本仍可阅读。
- Android 9、Android 12、Android 15/16 的核心流程通过测试。
- 自动化测试、关键手工回归和安全检查满足测试方案的发布门禁。
