# 语音朗读产品方案

| 项目 | 内容 |
|---|---|
| 版本 | `0.2 Draft` |
| 产品形态 | 类似微信公众号“听全文”的文章朗读 |
| 核心约束 | 本地免费、离线可用、支持后台播放、不上传知识库内容 |
| 首期范围 | Markdown 文档 |

## 1. 产品目标

为 RepoLeaf 增加“听全文”能力，让用户在通勤、步行、做家务或不便持续看屏幕时，也能连续消费知识库内容。

首期采用 Android 系统本地 TTS，不接入收费云服务，不需要服务端，不按字符计费。私有仓库正文和生成的音频均留在设备内。

### 1.1 成功标准

- 打开文章后，一次点击即可开始朗读。
- 首次播放无需等待整篇文章生成完毕，先合成前两段并尽快开播。
- 锁屏、切换应用和离开文章页面后仍可继续播放。
- 通知栏、锁屏和蓝牙耳机可以播放、暂停及切换前后段。
- 再次进入同一篇文章时，可以从上次位置继续。
- 中文、数字、金额、百分比和常见中英文缩写的朗读自然可懂。
- 开启飞行模式后，已同步文章仍可使用已安装的离线音色朗读。

### 1.2 非目标

- 首期不提供云端“真人级”音色、声音克隆和有声书配音。
- 首期不朗读 PDF、HTML、Excel；架构保留后续接入能力。
- 不保证所有品牌手机、所有 TTS 引擎的声音完全一致。
- 不提供跨设备同步播放进度和音频缓存。

## 2. 用户体验

### 2.1 入口

文章标题下方显示朗读卡片：

> 🎧 听全文 · 预计 12 分钟

点击后立即进入“准备中”，前两段合成完成即开始播放。已生成过的文章直接播放缓存。

### 2.2 迷你播放器

播放开始后，应用底部常驻迷你播放器：

- 当前文章标题与朗读进度；
- 播放/暂停；
- 当前倍速；
- 点击主体展开完整播放器；
- 关闭按钮停止播放，但保留进度。

用户返回目录或打开另一篇文章时，迷你播放器仍存在。点击它可回到正在朗读的文章。

### 2.3 完整播放器

完整播放器采用底部抽屉或独立页面，提供：

- 播放/暂停；
- 后退/前进 15 秒；
- 上一段/下一段；
- 可拖动进度条；
- `0.75× / 1.0× / 1.25× / 1.5× / 2.0×` 倍速；
- 音色选择；
- 当前段文字预览。

### 2.4 正文联动

- 朗读与正文阅读解耦：播放过程中用户可自由翻页、搜索和滚动，页面不会被强制定位或自动滚动。

### 2.5 后台播放

- 锁屏、回桌面、切换应用、进入目录时继续播放。
- 通知栏和锁屏显示文章标题、播放/暂停、上一段和下一段。
- 支持耳机按键和蓝牙设备的媒体控制。
- 来电或其他语音内容抢占音频焦点时暂停，焦点恢复后不擅自开播，由系统场景和用户操作决定。
- 拔出有线耳机或断开蓝牙时自动暂停，避免扬声器突然外放。

## 3. 内容朗读规则

| Markdown 内容 | 默认行为 |
|---|---|
| 标题 | 朗读，前后增加自然停顿 |
| 普通段落 | 按标点切成长度适中的语音段 |
| 有序/无序列表 | 逐项朗读，保留项目间停顿 |
| 引用 | 朗读正文，不朗读 `>` 等符号 |
| 链接 | 只读链接文字，不读完整 URL |
| 图片 | 默认跳过；可选读替代文字 |
| 代码块 | 默认跳过，可提示“代码块已跳过” |
| 行内代码 | 读取内容，去掉反引号 |
| 表格 | 默认播报“表格，共 X 行 Y 列”；可手动选择逐行朗读 |
| Front matter | 跳过 |

每段优先在句号、问号、感叹号、分号和逗号处分割，避免在数字、英文缩写或专有名词中间断开。

## 4. 中文友好处理

朗读前对文本做本地规范化：

- `11.1%` 转为“百分之十一点一”；
- `908亿元` 转为“九百零八亿元”；
- 日期、时间、货币和序号按中文习惯处理；
- URL、Markdown 标记、脚注符号和连续标点不直接朗读；
- 为 `AIDC`、`EBITDA` 等常见缩写提供内置发音词典；
- 中英混排时保留英文单词边界，减少逐字符误读；
- 用户可在设置中维护个人发音词典，后续版本支持仓库级词典。

## 5. 音色与设置

### 5.1 音色

可以支持音色选择，但音色由手机已安装的 TTS 引擎和离线语音包决定。应用扫描全部已安装 TTS 引擎，并按“引擎名称 · 音色名称”展示；应用只展示：

- 支持中文；
- 不依赖网络；
- 当前设备实际可用的音色。

每个音色显示引擎名称、音色名称、语言地区、质量等级及“离线”标识，并提供一句试听。默认自动选择质量最高的普通话离线音色。

若设备没有中文离线音色，应用给出明确引导，跳转系统 TTS 设置安装语音数据，不自动切换云端，也不上传文章。

### 5.2 用户设置

- 默认音色；
- 默认倍速；
- 图片替代文字是否朗读；
- 表格默认跳过或逐行朗读；
- 代码块跳过提示；
- 音频缓存上限，默认 500 MB；
- 清理全部朗读缓存；
- 个人发音词典。

## 6. 状态与异常

播放器状态包括：空闲、准备中、播放、暂停、缓冲、完成和失败。

| 场景 | 用户反馈与恢复方式 |
|---|---|
| 未安装可用 TTS 引擎 | 说明原因并提供系统设置入口 |
| 缺少中文离线语音 | 引导下载语音数据，安装后重新检测 |
| 某段合成失败 | 自动重试一次，仍失败则跳过并提示 |
| 存储空间不足 | 清理最旧缓存后重试；仍不足则提示用户 |
| 文档内容已更新 | 废弃旧音频缓存，从最接近的正文段重新开始 |
| 应用进程被系统回收 | 再次打开时恢复文章、段落、段内位置和倍速 |
| 文章被删除或仓库被移除 | 停止播放并保留可理解的错误提示 |

## 7. 隐私与成本

- 文本处理、语音合成、播放进度和音频缓存全部在本机完成。
- 不调用云 TTS API，因此没有按字符产生的服务费用。
- 不把私有仓库正文、文件路径或生成音频写入日志和崩溃上报。
- 生成音频放在应用私有缓存目录，不进入相册、音乐库或公共存储。
- 设置页明确显示当前 TTS 引擎、音色以及是否需要网络。

本地免费方案的主要代价是：不同手机的音色数量和中文自然度有差异。产品以“隐私、免费、稳定后台播放”为首要目标，通过文本规范化和可选音色尽量改善体验。

## 8. MVP 验收标准

- 在基准 Android 手机上，使用离线中文音色完整朗读一篇 30 分钟文章，无中断、无明显跳段。
- 热启动首次出声 P95 不超过 2 秒，冷启动 P95 不超过 5 秒。
- 锁屏、切换应用和进入目录后连续播放至少 60 分钟。
- 通知栏、锁屏及蓝牙耳机可完成播放/暂停和前后段控制。
- 应用退出再打开后，恢复到上次段落，段内位置误差不超过 3 秒。
- 倍速变化立即生效，不重新生成音频。
- 飞行模式下可朗读已同步文章，且无任何网络请求。
- 私有仓库文本与生成音频不离开应用私有空间。
- 缺少离线中文音色时不崩溃，能给出可操作的安装指引。

## 9. 迭代建议

1. `0.2.0`：基础分段、离线合成、后台播放、迷你播放器、进度恢复。
2. `0.2.1`：音色试听、中文数字和金融词典。
3. `0.2.2`：表格逐行朗读、个人发音词典、缓存管理和体验调优。
4. 后续评估：PDF/HTML 内容抽取、仓库级发音词典、可选的用户自带云 TTS。
